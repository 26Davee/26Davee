## Proyecto principal

### RepairFlow Service Desk

Version demo y saneada de un sistema para gestion de servicio tecnico. El
proyecto original pertenece a un entorno empresarial, por eso este repositorio
usa datos ficticios y no incluye informacion real, respaldos, evidencias,
credenciales ni bases de datos privadas.

- Aplicacion de escritorio: Python y PySide6.
- Portal web: Flask, HTML y CSS.
- Base de datos local: SQLite.
- Modulos: clientes, equipos, ordenes de servicio, estados, reportes y
  aprobaciones.
- Enfoque: separar una solucion real de empresa en una version presentable para
  portafolio.

Repositorio:

- [repairflow-service-desk](https://github.com/26Davee/repairflow-service-desk)

## Proyecto secundario

### OrderFlow Manager

Sistema full stack para gestionar pedidos, clientes, estados, imagenes y
estadisticas operativas desde una interfaz web.

- Frontend: React, Vite, CSS, Recharts.
- Backend: Node.js, Express, Prisma.
- Base de datos: PostgreSQL.
- Servicios externos: Cloudinary, Railway, Vercel.
- Organizacion: monorepo con `apps/web` y `apps/api`.

Repositorio:

- [orderflow-manager](https://github.com/26Davee/orderflow-manager)
